PROJECT PROGRESS SUMMARY - June 3, 2026

1. Authentication & Login Implementation:
   - Completed the login logic for all user roles using TaiKhoanDAO.
   - Implemented separate login servlets for better organization:
     * KhachHangLoginServlet.java
     * ShipperLoginServlet.java
     * CuaHangLoginServlet.java
   - Added strict role validation during login to ensure accounts can only access their respective areas (e.g., only 'Shipper' accounts can log in via ShipperLoginServlet).
   - Integrated HttpSession to store the 'user' (TaiKhoan) object upon successful authentication.

2. Profile API Development (Session-Based Flow):
   - Established a secure "My Profile" flow that retrieves data based on the logged-in session rather than URL parameters.
   - Implemented JSON-returning API servlets for all roles:
     * CuaHangProfileServlet.java (/api/shop/profile)
     * KhachHangProfileServlet.java (/api/user/profile)
     * ShipperProfileServlet.java (/api/shipper/profile)
   - These servlets fetch detailed profile information using the `getByAccountId` method from their respective DAOs (CuaHangDAO, KhachHangDAO, ShipperDAO).

3. Architectural Refactoring:
   - Organized controller classes into logical subfolders for better maintainability:
     * /controller/KhachHang/
     * /controller/Shipper/
     * /controller/CuaHang/
   - Standardized API URL mapping using the "/api/..." prefix to distinguish data endpoints from view pages.

4. DAO Enhancements:
   - Verified and utilized `getByAccountId` across all user-related DAOs.
   - Ensured consistency in role-string matching ("Khách hàng", "Shipper", "Cửa hàng") between Java code and SQL Server schema constraints.
